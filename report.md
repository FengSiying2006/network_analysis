### 第一部分：主要功能实现情况 (60分)

#### 1. 数据读取 (5分)
*   **演示操作**：展示 `data/network_data.csv` 文件内容。运行程序，控制台输出“成功读取 X 行数据”。
*   **核心代码**：`src/csv_reader.c` -> `read_csv()`。
*   **亮点说明**：
    *   使用了**倍增扩容算法** (`realloc`)，读取百万级行数据也能保持高效，避免频繁内存分配。
    *   健壮性处理：自动处理空行、缺失端口（ICMP默认0端口）等异常情况。

#### 2. 构建图 (13分)
*   **展示图数据结构 (5分)**：
    *   打开 `include/graph_builder.h`。
    *   讲解 `Graph` 结构体：
        *   **`csr_row_ptr` / `csr_col_idx`**：压缩稀疏行（CSR）格式，用于 $O(1)$ 快速查询出边。
        *   **`csc_col_ptr` / `csc_row_idx`**：压缩稀疏列（CSC）格式，用于 $O(1)$ 快速查询入边（这是本项目的一大亮点，支持双向高效查询）。
*   **构建图结构 (8分)**：
    *   讲解 `src/graph_builder.c` -> `build_graph()`。
    *   **核心逻辑**：
        1.  **IP 映射**：将字符串 IP 映射为连续整数 ID (0~N)，节省内存。
        2.  **排序聚合**：使用 `qsort` 对边进行排序，合并重复的会话（累加流量和时长）。
        3.  **前缀和构建**：一次遍历生成 CSR/CSC 数组，无需复杂的链表操作，内存连续，访问极快。

#### 3. 流量排序 (21分)
*   **演示操作**：
    *   在 Web 界面点击 **"Traffic Ranking" -> "Top Traffic Nodes"**，弹出表格。
    *   点击 **"Top HTTPS Nodes"**，展示 HTTPS 流量排名。
    *   点击 **"High Outbound Ratio"**，展示单向流量占比 > 80% 的节点。
*   **核心代码**：`src/sorting.c`。
*   **实现逻辑**：
    *   遍历 CSR/CSC 数组，分别累加 `out_traffic` 和 `in_traffic`。
    *   根据筛选条件（端口 443 或 `out/(in+out) > 0.8`）过滤节点。
    *   使用 `qsort` 进行快速降序排列。

#### 4. 路径查找 (21分)
*   **演示操作**：
    *   在 Web 界面输入源 IP 和 目的 IP。
    *   点击 **"Min Hops"**，图上高亮显示最短路径（跳数最少）。
    *   点击 **"Min Congest"**，图上高亮显示另一条路径（拥塞度最低）。
    *   **路径对比 (5分)**：展示两条路径的不同（例如：跳数少但拥塞，或者绕路但通畅）。
*   **核心代码**：`src/path.c`。
*   **算法说明**：
    *   **最小跳数**：使用 **BFS (广度优先搜索)**，保证找到边数最少的路径。
    *   **最小拥塞**：使用 **Dijkstra 算法**，边权重为 `流量/时长`，使用优先队列优化。

### 第四部分：程序规范性与特色 (14分)

#### 7. 程序规范性 (7分)
*   **模块化设计**：项目分为 `include` (头文件), `src` (C源码), `python` (脚本), `web` (前端) 四大目录，结构清晰，高内聚低耦合。
*   **数据结构合理**：
    *   底层使用 **CSR + CSC** 压缩稀疏矩阵，内存占用少，缓存命中率高，性能远超链表。
    *   IP 映射使用 **二分查找 (Binary Search)**，查询效率 $O(\log N)$。
*   **注释完整**：所有关键函数（如 `build_graph`, `dijkstra`）均有详细注释。

#### 8. 设计特色和创意 (7分)
*   **全栈架构创新**：摒弃传统的控制台或 Tkinter，采用 **C (高性能计算) + Python Flask (Web后端) + Vis.js (前端可视化)** 的现代架构。
*   **算法创新**：
    *   实现了 **社区发现 (Community Detection)** 算法，自动识别网络群组。
    *   实现了 **僵尸网络启发式检测**，自动标记高危节点（红色闪烁）。
*   **实时性**：实现了 **自动抓包 + 实时轮询** 的端到端监控系统，不仅仅是离线分析工具。
*   **地理位置增强**：结合 IP 地理位置库，在图上展示节点的物理位置（如 "Beijing, China"）。
